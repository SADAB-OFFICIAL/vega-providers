# Vega Providers API

## Project Overview
This is a backend API server for the Vega app that provides streaming content from multiple providers. The project compiles TypeScript provider modules into JavaScript and serves them via a REST API.

**Current Status:** ✅ Running and accessible
**API URL:** Available via Replit webview
**Last Updated:** November 2, 2025

## Project Architecture

### Structure
- **`providers/`**: TypeScript source files for 40+ streaming providers
- **`dist/`**: Compiled JavaScript providers (generated by build process)
- **`server.js`**: Main Express API server
- **`dev-server.js`**: Development server for local testing (port 3001)
- **`build-simple.js`**: Build script that compiles and minifies TypeScript

### Key Components
1. **Provider Context** (`providers/providerContext.ts`): Shared utilities including axios, cheerio, crypto functions, and extractors
2. **Provider Modules**: Each provider has catalog, meta, posts, stream, and optional episodes files
3. **API Server**: Express server providing REST endpoints for posts, metadata, and streams

## Environment Setup

### Dependencies
- **Node.js 20**: Primary runtime
- **TypeScript**: For provider source files
- **Express**: Web server framework
- **Axios**: HTTP client
- **Cheerio**: HTML parsing
- **Terser**: JavaScript minification

### Build Process
The build process (`npm run build`):
1. Compiles all TypeScript files from `providers/` to JavaScript
2. Organizes compiled files in `dist/` directory
3. Minifies JavaScript files and removes console logs
4. Reduces bundle size by ~41.5%

## API Endpoints

### Base URL
The API is accessible via the Replit webview URL.

### Available Endpoints

#### 1. Get Posts (Homepage/Catalog)
```
GET /api/posts/:provider?filter=<filter>&page=<page>
```
Example: `/api/posts/vega?filter=/category/popular&page=1`

#### 2. Search Posts
```
GET /api/posts/:provider?searchQuery=<query>&page=<page>
```
Example: `/api/posts/uhd?searchQuery=avengers&page=1`

#### 3. Get Metadata
```
GET /api/meta/:provider?link=<encoded_link>
```
Example: `/api/meta/vega?link=https%3A%2F%2Fexample.com%2Fmovie`

#### 4. Get Stream Links
```
GET /api/stream/:provider?link=<encoded_link>&type=<movie|series>
```
Example: `/api/stream/vega?link=https%3A%2F%2Fexample.com%2Fstream&type=movie`

## Development Workflow

### Building the Project
```bash
npm run build
```
This compiles TypeScript providers to JavaScript in the `dist/` directory.

### Running the Production Server
The server automatically starts via the workflow configuration:
```bash
node server.js
```
Runs on port 5000 (bound to 0.0.0.0 for Replit accessibility)

### Local Development Server
For local testing with the mobile app:
```bash
npm run auto
```
Runs on port 3001 with auto-rebuild capabilities.

## Technical Notes

### Crypto Implementation
The project originally used `react-native-aes-crypto` which is not compatible with Node.js. It was replaced with a custom Aes wrapper using Node.js's built-in `crypto` module:
```typescript
const Aes = {
  sha1: async (input: string): Promise<string> => {
    return crypto.createHash('sha1').update(input).digest('hex');
  }
};
```
